## Purpose

Lean project constitution.

- project identity

### Identity
- project name
- codebase/module names
- primary stack

### Conventions
- coding rules
- ownership rules
- testing expectations
- deployment or environment assumptions

### Invariants
- non-negotiable architecture rules
- authority boundaries
- prohibited patterns
